## 1. 项目概述

本项目是一个面向考研学生的**专业化 RAG（检索增强生成）知识问答系统**，旨在为考研学生提供精准的专业课知识检索与问答服务。

### 核心能力

| 模块 | 功能描述 |
|------|----------|
| **多Agent管线** | Router(三分类) → Math/English/Politics Specialist → 回答 |
| **知识检索 (RAG)** | 纯Python关键词匹配，从110个知识库文档中精准检索 |
| **幻觉检测** | LLM-as-Judge通用领域常识版，50题Benchmark 100%准确率 |
| **Skill系统** | 6个格式/内容型Skill自动加载，侧边栏Toggle切换 |
| **多用户** | SHA256登录/注册，SQLite user_id数据隔离 |
| **自学习** | 增量经验追加，不可变约束 + 动态经验库双层架构 |

## 3. 核心亮点

### 3.1 防幻觉机制

- **相似度阈值过滤**：只保留相似度 > 0.02 的检索结果
- **保底策略**：即使无结果满足阈值，强制返回分数最高的文档
- **Prompt 约束**：系统提示词明确要求"只基于参考资料回答，不知道则明确告知"

### 3.2 多文档检索能力

- 支持多本专业课资料同时加载
- 按相似度从高到低排序
- 显示 Top-K 参考来源及相似度分数

### 3.3 智能 Chunk 切分系统

- **文档类型识别**：自动检测7种文档类型（知识点、考纲、制度、技术、问答、考试、通用）
- **手动模式选择**：支持一键选择文档类型，对应不同切分策略
- **可折叠说明**：策略说明默认收起，选择后突出显示当前策略
- **参数可调**：Chunk 大小（200-1000字符）和 Overlap 重叠（0-100字符）可自由调节

### 3.4 界面优化

- 统计指标合并为一行显示（6个指标紧凑排列）
- 检索模式帮助说明（TF-IDF/BM25/Hybrid 三种模式选择指南）
- UTF-8 编码优先读取，解决中文乱码问题

## 11. 考研AI幻觉率约束系统 (2026-05)

### 系统概述

基于 LLM-as-Judge 架构的幻觉检测Harness，严格按照"上下文约束"原则评估回答的事实性。

### 核心功能

| 功能 | 描述 |
|------|------|
| **声明提取** | 将回答拆解为独立的客观Claims |
| **交叉验证** | 逐一检查Claim与Context的一致性 |
| **专业常识宽容** | 标准数学公理/等价术语不记为有害幻觉 |
| **有害幻觉判定** | 仅判定捏造考研分值/频率/历史背景的幻觉 |

### Math-Aware 评估逻辑

| 类别 | 示例 | 处理 |
|------|------|------|
| 严格支持 | 回答直接来自Context | ✅ 通过 |
| 专业常识拓展 | "可逆矩阵=满秩矩阵"、"微分是线性运算" | ✅ 宽容通过 |
| 有害幻觉 | "每年必考"、"占30分"、"牛顿发明" | ❌ 判定为幻觉 |

### 测试结果

| 指标 | 数值 |
|------|------|
| 总测试数 | 15 |
| 有害幻觉检测率 | 46.7% (7/15) |
| 专业常识识别准确率 | 100% (5/5) |

### 使用方式

```bash
# 单次测试
python hallucination_detector.py -q "问题" -c "上下文" -r "回答"

# 批量测试
python hallucination_detector.py -b test_data/hallucination_tests.json
```

### 文件清单

| 文件 | 说明 |
|------|------|
| `hallucination_detector.py` | 幻觉检测核心模块 |
| `test_data/hallucination_tests.json` | 考研领域测试集 (15题) |
| `benchmarks/hallucination_report.md` | 测试报告 |
| `templates/math_tutor_prompt.md` | 执行AI系统Prompt模板 |

## 12. 执行AI约束系统 (2026-05)

### 系统架构

```
执行AI (Math Tutor) ← 严格System Prompt ← app.py
                           ↓
                    约束规则：
                    1. 信息溯源
                    2. 纯粹数学
                    3. 禁止考情预测
                    4. 禁止无关延伸
```

### 约束规则

| 规则 | 描述 | 违规示例 |
|------|------|----------|
| **信息溯源** | 必须基于Context作答 | 凭借内部知识回答 |
| **纯粹数学** | 只谈数学原理 | 谈历史/应用 |
| **禁止考情预测** | 不猜测考情 | "每年必考" |
| **禁止无关延伸** | 不补充Context外内容 | "牛顿发明" |

### 集成方式

app.py 中的 `call_llm()` 函数已更新 system prompt，执行AI现在会：
- 严格基于RAG检索结果回答
- 不预测考研考情
- 不延伸历史/应用背景

### 文件

- `templates/math_tutor_prompt.md` - Prompt模板
- `app.py` - 已集成严格Prompt

## 25. LaTeX渲染修复 + 缓存刷新 (2026-05-16)

### LaTeX渲染

**根因**：`st.markdown(unsafe_allow_html=True)` 将 `$...$` 和 `$$...$$` 当作文本输出。

**修复**：知识库展开、复习挑战内容改用 `st.markdown()` 直接解析，MathJax 自动渲染 LaTeX。

| 位置 | 之前 | 之后 |
|------|------|------|
| 知识库搜索结果 | `<div class='scrollable-text'>{text}</div>` unsafe_html | `st.markdown(text)` |
| 知识库浏览列表 | 同上 | 同上 |
| 复习挑战展开 | 同上 | 同上 |
| 生成复习题 prompt | 未要求 LaTeX | 明确要求 $...$/$$...$$ |

### 缓存刷新

**问题**：新增知识库文件后 `@st.cache_data` 不自动刷新，始终显示100文档。

**修复**：侧边栏新增 `🔄 刷新知识库` 和 `🔌 重连API` 按钮，调用 `st.cache_data.clear()` + `st.rerun()`。
